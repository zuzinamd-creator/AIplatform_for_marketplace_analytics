# SKU Drilldown (глубокий разбор товара)

Цель: дать продавцу **понятный разбор “почему товар убыточен”** и **что ухудшилось относительно прошлого периода** на уровне одного SKU, без изменения финансовых инвариантов (append-only ledger, governed projections, RLS).

## Маршруты (frontend)

- `/app/economics/sku/:sku` — drilldown по SKU

## API (backend)

- `GET /api/v1/analytics/sku-economics/sku/{sku}/drilldown`
  - query: `marketplace`, `start`, `end`, `semantics_version`
  - ответ: `SkuDrilldownResponse`

## Что показываем продавцу

- **Тренды**: выручка, валовая прибыль, маржа, возвраты, логистика, реклама, штрафы, остатки (если есть снапшоты склада)
- **Объяснения**:
  - “Почему товар убыточен”
  - “Что сильнее всего съедает прибыль”
  - “Что ухудшилось относительно прошлого периода” (при включенном сравнении)
- **Доверие к данным**:
  - полнота (completeness score)
  - предупреждения (нет себестоимости / расхождение выплат / аномалии)
  - “ИИ может быть неточным, потому что…”

## Принципы

- Никакой “магии” ML: объяснения — это **детерминированная агрегация** из governed проекций.
- Сверка выплат и прибыль — разные сущности: payout — денежный поток, profit — P&L.
